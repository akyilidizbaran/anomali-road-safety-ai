# 6. Zaman Planlaması

## Resmi Beklenti

Geliştirme ve test süreçlerini içeren zaman planı verilmelidir. Proje takip aracı ve yöntemi açıklanmalıdır.

## Takip Yöntemi

* Yöntem: Kanban + haftalık sprint.
* Araç: GitHub Projects, Trello veya Notion.
* Her sprint çıktısı: çalışan modül, test sonucu veya rapor bölümü.

## Önerilen Takvim

| Dönem | İş Paketi | Çıktı |
|---|---|---|
| Hafta 1 | Mimari ve API contract | Veri akışı, event JSON, görev dağılımı |
| Hafta 2-3 | Mobil kamera + edge bağlantı | Camera preview, frame aktarımı |
| Hafta 4-5 | Araç tespiti + tracking | Bbox, track ID, target vehicle |
| Hafta 6-7 | Plaka/OCR + evidence | OCR, evidence kartları |
| Hafta 8-9 | Sahne, şerit, hız | Risk sinyalleri |
| Hafta 10 | QoD/Number Verification adapter | Mock/gerçek servis ayrımı |
| Hafta 11 | Test ve optimizasyon | FPS, latency, model metrikleri |
| Hafta 12 | Rapor ve demo | Final rapor, sunum, demo script |
