# PDR/ÖTR - Ön Tasarım Raporu Başlık Haritası

Resmi Ön Tasarım Raporu şablonundaki başlıklar:

1. Proje Özeti
2. Takım Şeması
3. Sistem Ön Tasarımı
   * 3.1 Genel Sistem Mimarisi ve Veri Akış Şeması
   * 3.2 Yapay Zeka Modeli ve Çıkarım Tasarımı
   * 3.3 Yazılım Mimari Tasarımı
4. Yöntem
5. Özgünlük
6. Zaman Planlaması
7. Proje Rapor Düzeni ve Kaynakça

## PDR/ÖTR Yazım Stratejisi

Ön tasarım raporunda amaç bitmiş sistemi ispatlamak değil, yapılabilir ve mantıklı bir mühendislik tasarımı göstermektir. Bu nedenle:

* “Tasarlanmıştır”, “hedeflenmektedir”, “planlanmaktadır” dili kullanılmalıdır.
* Henüz ölçülmemiş metrikler hedef olarak yazılmalıdır.
* Gerçek 5G API bağlantısı yoksa mock/stub adapter yaklaşımı açıkça belirtilmelidir.
* Normal/kritik mod ve seçici uzman model yaklaşımı sistemin ana verimlilik gerekçesi olarak kullanılmalıdır.
* QoD’nin seçici kullanımı özgünlük ve kaynak verimliliği başlıklarında vurgulanmalıdır.
* Rapor 3-10 sayfa sınırında kalacağı için bu klasördeki detaylar rapora özetlenerek aktarılmalıdır.
