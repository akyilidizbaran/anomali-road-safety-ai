# PCR/FTR - Final Tasarım Raporu Başlık Haritası

Resmi Final Tasarım Raporu şablonundaki başlıklar:

1. Proje Özeti
2. Veriseti Oluşturulması
3. Yapay Zeka Çözümü
   * 3.1 Problemin Analizi
   * 3.2 Çözüm Mimarisi
   * 3.3 Çözüm Detayları
4. Çözümün Sınanması
5. Kaynakça

## PCR/FTR Yazım Stratejisi

Final raporda tasarım değil kanıt beklenir. Bu nedenle:

* Veri seti ve split oranları açık yazılmalıdır.
* Model metrikleri tablo/grafik halinde verilmelidir.
* FPS, latency ve sistem performansı ölçülmelidir.
* Evidence sistemi “çözümümüze neden güveniyoruz?” sorusunun parçası yapılmalıdır.
* Mock kalan servisler saklanmamalı, açıkça belirtilmelidir.
