# System ve Settings Ekranları

## System Ekranı

Amaç: Pipeline sağlığını göstermek.

Göstergeler:

* Kamera aktif mi?
* Edge bağlantısı var mı?
* Backend ayakta mı?
* Model server çalışıyor mu?
* FPS.
* Ortalama latency.
* Normal/kritik mod.
* QoD mock/gerçek.
* Number Verification mock/gerçek.
* Evidence storage durumu.
* Aktif model versiyonları.
* Son hata mesajı.

## Settings Ekranı

Amaç: Demo ve model davranışını ayarlamak.

Ayarlar:

* Single-target / multi-target.
* Risk threshold.
* OCR threshold.
* Lane detection açık/kapalı.
* Speed estimation açık/kapalı.
* Driver analysis açık/kapalı.
* QoD mock açık/kapalı.
* Evidence limit.
* Çözünürlük.
* Frame skipping.
* Model kalite modu: fast / balanced / accurate.

## Sorulacak Noktalar

* Settings ekranı jüri demosunda erişilebilir olacak mı?
