# Araştırma 10 - Evidence Sistemi ve Açıklanabilirlik

## Amaç

Riskli olayları sadece alarm olarak değil, sonradan incelenebilir kanıt paketleri olarak kaydetmek.

## Alt Başlıklar

* Evidence kartı tasarımı.
* Son 10 olay.
* Görüntü kesiti saklama.
* Overlay’li screenshot.
* JSON metadata.
* Kanıt göster/detay göster.
* LLM açıklama katmanı.
* LLM’nin karar verici olmaması.
* Event ID, timestamp, bbox, confidence, model version.
* Backend storage.
* Evidence integrity.
* Denetlenebilir karar zinciri.
* Mobil UI/UX.
* Rapor için örnek evidence çıktıları.

## Çıktı

Event JSON şeması ve evidence storage standardı.
